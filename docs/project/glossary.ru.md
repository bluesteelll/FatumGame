# Глоссарий

> Ключевые термины, используемые в документации FatumGame. Если термин встречается без объяснения на другой странице, вы найдёте его здесь.

---

## Основные концепции

### Barrage
Плагин интеграции Jolt Physics. Оборачивает `BodyInterface` Jolt, слушателей контактов, рейкасты и констрейнты в UE-дружественные API. Центральный класс -- `UBarrageDispatch` (`UTickableWorldSubsystem`).

### SkeletonKey
64-битный идентификатор сущности, используемый во всех подсистемах (Flecs, Barrage, Render Manager). Старший ниббл кодирует тип сущности:

| Ниббл | Тип | Описание |
|-------|-----|----------|
| `0x3` | `GUN_SHOT` | Снаряды |
| `0x4` | `BAR_PRIM` | Общие примитивы Barrage (предметы, разрушаемые) |
| `0x5` | `ART_ACTS` | Акторы (персонажи) |
| `0xC` | `ITEM` | Сущности предметов |

### FBarragePrimitive
Shared pointer (`TSharedPtr<FBarragePrimitive>`), представляющий одно физическое тело Jolt. Содержит `SkeletonKey` (`KeyIntoBarrage`), атомарный `FlecsEntityId` для обратной привязки и очередь команд позиции/поворота.

### Prefab
Сущность Flecs, созданная через `World.prefab()`, хранящая общие статические данные (макс. HP, урон, ссылка на меш). Экземпляры наследуют от prefab через отношение `IsA`, не платя памятью за наследуемые компоненты.

### Экземпляр (Entity)
Сущность Flecs, наследующая от prefab и добавляющая per-entity изменяемые данные (текущее HP, ключ физического тела, слот ISM). Каждый снаряд, предмет и разрушаемый объект в мире -- это экземпляр.

### ISM (Instanced Static Mesh)
`UInstancedStaticMeshComponent` UE. Все ECS-сущности с одинаковым мешом+материалом рендерятся через один ISM, устраняя overhead draw call на каждую сущность. Управляется `UFlecsRenderManager`.

### MPSC-очередь
Очередь с множеством продюсеров и одним потребителем (`TQueue<T, EQueueMode::Mpsc>`). Используется для всех межпоточных передач данных, где важен порядок. Множество вызывающих с game thread могут ставить в очередь одновременно; sim thread дренирует последовательно.

## Потоки

### Game Thread
Основной поток UE. Обрабатывает ввод, камеру, ISM-рендеринг, UI и VFX Niagara. Никогда не мутирует состояние мира Flecs напрямую -- все мутации идут через `EnqueueCommand`.

### Поток симуляции (Sim Thread)
`FSimulationWorker` -- выделенный поток `FRunnable`, тикающий на ~60 Гц. Владеет миром Flecs и управляет физикой Jolt. Единственный поток, которому разрешено вызывать API Flecs (кроме параллельного `progress()`).

### Рабочие потоки Flecs
Фоновые потоки, порождаемые Flecs во время `world.progress()` для параллельного выполнения систем. Количество = ядра CPU - 2. Должны вызывать `EnsureBarrageAccess()` перед обращением к API Jolt.

### EnqueueCommand
`UFlecsArtillerySubsystem::EnqueueCommand(TFunction<void()>)` -- основной путь мутаций game -> sim thread. Лямбды дренируются в начале каждого тика симуляции.

### FLateSyncBridge
Атомарный мост с семантикой последнего значения для данных, где важно только самое свежее (направление прицеливания, позиция камеры). Семантика перезаписи -- нет очереди, нет обратного давления.

### FSimStateCache
16-слотовый, выровненный по cache-line, lock-free кеш для скалярных данных sim -> game (здоровье, боезапас оружия, ресурсы способностей). Упаковывает значения в атомарные `uint64` для чтений без конкуренции.

## ECS

### Компонент
USTRUCT, зарегистрированный в Flecs, содержащий данные, прикреплённые к сущностям. Компоненты могут быть статическими (на prefab) или instance (per-entity).

### Тег
USTRUCT нулевого размера (`sizeof == 0`), используемый для классификации и фильтрации. Теги не имеют данных -- служат булевыми маркерами (напр., `FTagDead`, `FTagProjectile`).

### Система
Система Flecs, выполняемая каждый тик во время `world.progress()`. Регистрируется через `World.system<T>()`. Системы выполняются в порядке регистрации в рамках каждой фазы.

### Observer
Observer Flecs, срабатывающий реактивно на конкретное событие (напр., `flecs::OnSet`). Только `DamageObserver` использует этот паттерн в FatumGame.

### Пара столкновений
Временная сущность Flecs, создаваемая на каждый физический контакт. Содержит `FCollisionPair` (ID EntityA, EntityB) и один или несколько тегов столкновений. Уничтожается `CollisionPairCleanupSystem` в конце каждого тика.

### Отложенная операция
Мутация Flecs (set, add, remove), которая ставится в staging во время выполнения системы и применяется в следующей точке слияния. Источник нескольких [критических багов](../guidelines/ecs-best-practices.md).

## Физика

### Jolt
Физический движок за Barrage. Обрабатывает симуляцию твёрдых тел, обнаружение столкновений, рейкасты и констрейнты. Все вызовы Jolt идут через API Barrage, не через прямые API Jolt.

### Слой объектов
Механизм фильтрации столкновений Jolt. Каждое тело принадлежит слою, определяющему, с какими другими слоями оно сталкивается:

| Слой | Используется для |
|------|-----------------|
| `MOVING` | Активные динамические тела (фрагменты, двери) |
| `NON_MOVING` | Статическая геометрия мира, целые разрушаемые |
| `PROJECTILE` | Снаряды игрока |
| `ENEMYPROJECTILE` | Снаряды врагов |
| `DEBRIS` | Мёртвые/спящие тела (без реакции на столкновения) |
| `CHARACTER` | Физические капсулы игрока и NPC |
| `SENSOR` | Триггерные объёмы, зоны подбора |
| `CAST_QUERY` | Геометрия только для рейкастов |

### Tombstone
Безопасный паттерн уничтожения физических тел. Вместо немедленного удаления (что вызывает краш Jolt во время `StepWorld`), устанавливается слой `DEBRIS` (мгновенное отключение коллизий) и вызывается `SuggestTombstone()` (отложенная очистка через ~19 секунд).

### FDebrisPool
Кольцевой буфер пула предаллоцированных динамических боксовых тел Barrage. Используется системой разрушаемых для избежания аллокации тел в рантайме при фрагментации. Тела циклически переключаются между слоями `DEBRIS` (спящий) и `MOVING` (активный).

### Констрейнт
Физический констрейнт Jolt, связывающий два тела. Используемые типы: Fixed (фрагменты разрушаемых), Hinge (двери), Distance (цепи). Создаётся через `FBarrageConstraintSystem` или `UFlecsConstraintLibrary`.

## Data Assets

### UFlecsEntityDefinition
Мастер data asset. Ссылается на все профили, определяющие тип сущности. Создаётся в Content Browser и назначается спаунерам.

### Профиль
Подкласс `UObject` (EditInlineNew), встроенный в `UFlecsEntityDefinition`. Каждый профиль вносит конкретные данные в prefab сущности (напр., `UFlecsHealthProfile` -> `FHealthStatic`, `UFlecsPhysicsProfile` -> конфиг тела Barrage).

### UFlecsItemDefinition
Отдельный data asset для метаданных предмета (отображаемое имя, иконка, размер стака, размер в сетке, редкость). Ссылается из `UFlecsEntityDefinition` для предметов.

## Рендеринг

### Интерполяция рендера
Техника линейной интерполяции между предыдущей и текущей физическими позициями с использованием подтиковой альфы, обеспечивающая плавный визуал при любом фреймрейте несмотря на физический тик 60 Гц.

### Альфа (интерполяции)
`Alpha = (CurrentTime - LastSimTickTime) / SimDeltaTime`. Значение в `[0, 1]`, представляющее, как далеко текущий кадр рендера находится между двумя тиками симуляции. Alpha = 0 означает использование предыдущей позиции; Alpha = 1 -- текущей.

### PivotOffset
Корректирующий вектор, применяемый к трансформам ISM. Меши UE могут не иметь pivot в центре bounding box, поэтому `PivotOffset = -Bounds.Origin` поворачивается и добавляется к позиции каждого экземпляра.

### FEntityTransformState
Per-entity состояние рендера: `PrevPosition`, `CurrPosition`, `PrevRotation`, `CurrRotation`, `LastUpdateTick`, `bJustSpawned`. Используется менеджером рендера для интерполяции.

## Замедление времени

### FTimeDilationStack
Стек приоритетов на game thread, управляющий множеством источников замедления времени. Правило разрешения: минимум побеждает (наименьший `DesiredScale` среди активных записей вступает в силу).

### RealDT
Wall-clock дельта времени: разница `FPlatformTime::Seconds()` между тиками. Не затронут замедлением времени.

### DilatedDT
`RealDT * ActiveTimeScale`. Используется физикой (`StepWorld`) и ECS (`progress`).

### VelocityScale
`1.0 / ActiveTimeScale` когда `bPlayerFullSpeed = true`. Применяется к локомоции персонажа, чтобы игрок двигался с реальной скоростью, пока мир в замедлении.

## UI

### Плагин FlecsUI
Плагин, предоставляющий базовые классы для lock-free, model/view UI-панелей: `UFlecsUIPanel` (активируемый CommonUI), `UFlecsUIWidget` (обычный UUserWidget), `UFlecsContainerModel`, `UFlecsValueModel`.

### Тройной буфер (TTripleBuffer)
Lock-free примитив тройной буферизации. Sim thread пишет снимки через `WriteAndSwap()`; game thread читает последний полный снимок. Используется для синхронизации состояния контейнеров.

### UFlecsActionRouter
Пользовательский маршрутизатор ввода, управляющий видимостью курсора, захватом мыши и блокировкой look/move ввода при активных UI-панелях. Заменяет необходимость в `ActionDomainTable` CommonUI.

### Оптимистичный Drag-Drop
Паттерн UI инвентаря, при котором предметы перемещаются визуально сразу при перетаскивании, без ожидания подтверждения от sim thread. Если sim thread отклоняет операцию, UI откатывается.

## Система оружия

### Устройство быстрой зарядки (Quick-Load Device)

Предмет, который пакетно загружает патроны во внутренний магазин оружия. Настраивается через `UFlecsQuickLoadProfile` в определении сущности. Два подтипа: спидлоадер и обойма (stripper clip).

### Спидлоадер (Speedloader)

Устройство быстрой зарядки, которое одновременно вставляет все патроны в барабан револьвера. Требует полностью пустой магазин (`bRequiresEmptyMagazine = true`).

### Обойма (Stripper Clip)

Устройство быстрой зарядки, которое подаёт патроны во внутренний магазин (напр., винтовка с продольно-скользящим затвором). Не требует пустого магазина -- можно дозарядить частично заряженное оружие.

### Кольцо дроби (Pellet Ring)

Концентрическое кольцо дробин в паттерне разброса дробовика. Определяется через `FPelletRing` в `UFlecsWeaponProfile`. Каждое кольцо имеет радиус, количество дробин и значения джиттера. Несколько колец создают реалистичные паттерны дробовика (напр., внутреннее плотное кольцо + внешнее широкое кольцо). Когда `PelletRings` не пусто, оно переопределяет `ProjectilesPerShot`.

### Калибр (Caliber)

Именованный размер боеприпасов (напр., "9x19mm", "12ga"). Регистрируется в `UFlecsCaliberRegistry`. Оружие и типы боеприпасов ссылаются на калибры по `FName` для обеспечения совместимости.

### Профиль магазина (Magazine Profile)

`UFlecsFlecsMagazineProfile` -- определяет внутренний магазин оружия: ёмкость, калибр, совместимые типы боеприпасов и поведение перезарядки (поштучная vs полная). Создаёт ECS-компонент `FMagazineStatic` на prefab оружия.
